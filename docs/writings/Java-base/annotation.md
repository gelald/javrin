# 注解的原理

注解的原理可以分为三个层次来理解：注解是什么、注解的核心属性、注解如何使用

## 注解是什么

注解是用于描述数据的数据，称为“元数据标签”，本身不包含逻辑，它只是负责给数据打上各种标签

## 注解的核心属性

注解定义通过 `@interface` 来实现，注解的核心属性有：注解的生命周期、注解的适用范围

注解的生命周期是通过元注解 `@Retention` 标记，可以分为：
- `SOURCE`（仅保留在源文件，编译器编译后就消失），比如 `@Override`
- `CLASS`（编译后仍会保留，但是无法在运行程序时获取）
- `RUNTIME`（运行时可以通过反射获取，最常用），比如 `@Autowired`

注解的适用范围是通过元注解 `@Target` 标记，常见的有：
- `TYPE`（可以标记在类上）
- `METHOD`（可以标记在方法上）
- `FIELD`（可以标记在字段上）
- `ANNOTATION_TYPE`（可以标记在注解上）
它们互相不互斥，可以同时指定

## 注解的使用

注解自身没有逻辑，是提供给编译器或者框架来使用

比如常见的 `@Autowired` 本身不会完成依赖注入的工作，它通过标记字段告诉 Spring 框架，这个字段需要特殊处理，由 Spring 的 `AutowiredAnnotationBeanPostProcessor` 来扫描注解并参与依赖注入的工作
